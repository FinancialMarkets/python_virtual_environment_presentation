Virtual Environments

Each python package has its own set of dependencies, and so if you install many packages these dependencies will inevitably conflict. The solution is to create an environment for each project, and only install packages necessary for that project.

Different Python Versions

Virtual environments also allow you to have separate versions of Python itself (a unique version in each environment). So, if for some reason you need an early version of Python 3, or even Python 2, you can run it in a virtual environment.

Python venv vs Anaconda

Python 3 has the ability to create virtual environments built-in. However, Anaconda Python offers a GUI for creating environments and installing packages, and beginners often prefer this option.

Anaconda Python

Anaconda Python is a version of Python (distribution) that is maintained by Anaconda Inc. They have paid versions, and a free version.

Anaconda doesn't give you any functionality that venv and pip don't already provide. It is just an easier to use. Though the downside is it can be slow---easier to use is more complex to implement.

Python venv

Python itself provides the venv module for creating virtual environments. The module is lightweight, and so requires a bit more knowledge from the user.

venv doesn't allow creating environments with different versions on Python however. To do that you can use the virtualenv package.

	    
	      python3 -m venv /path/to/new/virtual/environment
